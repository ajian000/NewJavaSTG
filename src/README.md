# src 目录说明

`src` 目录包含了项目的所有源代码文件，按照功能和模块进行了组织。

## 目录结构

- **Main/**: 包含主程序入口文件 `Main.java`，负责启动游戏。
- **stg/**: 包含游戏的核心框架和基础组件，
  - **base/**: 基础UI和输入处理组件，
  - **game/**: 游戏核心逻辑，
    - **bullet/**: 子弹相关类，
    - **enemy/**: 敌人相关类，
    - **event/**: 游戏事件类，
    - **item/**: 道具相关类，
    - **laser/**: 激光相关类，
    - **obj/**: 游戏对象基类，
    - **player/**: 玩家相关类，
    - **stage/**: 关卡相关类，
    - **ui/**: 游戏UI组件。
  - **util/**: 工具类，
    - **math/**: 数学工具类。
- **user/**: 包含用户自定义的游戏内容，
  - **bullet/**: 自定义子弹，
  - **enemy/**: 自定义敌人，
  - **item/**: 自定义道具，
  - **laser/**: 自定义激光，
  - **player/**: 自定义玩家，
  - **stage/**: 自定义关卡。

## 开发指南

1. **添加新功能**: 请根据功能类型，在相应的子目录中创建新的类文件。
2. **遵循命名规范**: 类名使用 PascalCase，方法和变量使用 camelCase。
3. **代码风格**: 保持代码风格一致，使用适当的缩进和注释。
4. **测试**: 在添加新功能后，请确保进行适当的测试，以确保游戏的稳定性。

## 注意事项

- 不要修改 `stg` 目录中的核心框架代码，除非你完全理解其功能和影响。
- 自定义内容请放在 `user` 目录中，以便于管理和更新。
- 确保所有新增的类都遵循项目的接口规范，特别是实现了相应的接口。
