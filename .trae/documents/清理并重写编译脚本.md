## 任务计划

### 1. 清理 src 目录中的 .class 文件
- 搜索并删除 src 目录中所有 .class 文件
- 验证清理结果

### 2. 重写编译脚本
- 创建新的 compile.bat 脚本
- 脚本功能：
  - 清理 bin 目录中的旧编译文件
  - 递归编译 src 目录中的所有 Java 文件到 bin 目录
  - 使用 UTF-8 编码
  - 正确设置类路径
  - 提供清晰的编译进度信息

### 3. 验证新脚本
- 运行新脚本编译项目
- 检查 bin 目录是否正确生成所有 .class 文件
- 验证项目能否正常运行

## 技术细节

### 清理命令
使用 `find` 或 `del` 命令递归删除 src 目录中的 .class 文件

### 编译命令
使用 `javac` 命令的 `-d` 参数指定输出目录为 bin，使用 `-sourcepath` 指定源文件路径，并使用通配符递归编译所有 Java 文件

### 脚本结构
- 清理部分：删除旧编译文件
- 编译部分：按包依赖顺序编译
- 验证部分：检查编译结果
- 运行部分：可选运行项目

此计划将确保项目编译过程更加自动化和可维护，同时保持代码和编译产物的分离。
